Liver Disease Image Classifier

Context

Liver disease is a serious medical condition with various causes and types. Early detection can significantly improve treatment outcomes. Medical imaging, particularly liver ultrasound images, is an effective tool for diagnosis. However, manual analysis is time-consuming and prone to human error. This project uses deep learning models to automate liver disease classification from ultrasound images.

Objective

To build and compare two deep learning modelsa custom CNN and a VGG16-based transfer learning modelto classify liver ultrasound images into binary classes (diseased vs. healthy). The goal is to improve classification accuracy while minimizing overfitting and resource consumption (RAM/CPU).

Dataset

https://universe.roboflow.com/roboflow-100/liver-disease/

  • The dataset consists of liver ultrasound images, organized into subfolders by class:

    • train/ Training images grouped in folders (one folder per class).

    • test/ Validation images grouped similarly.

  • Each image is a .png or .jpg file representing an ultrasound scan. The folder names act as class labels. This is a binary classification task:

    • 0: Healthy liver
    • 1: Liver disease present

import random

def copy_subset_images(src_folder, dst_base, max_per_class=300):
    for fname in os.listdir(src_folder):
        if not fname.lower().endswith(('.jpg', '.jpeg', '.png')):
            continue

        match = re.match(r"^(\d+)_", fname)
        if not match:
            continue

        class_label = match.group(1)
        dst_folder = os.path.join(dst_base, class_label)
        os.makedirs(dst_folder, exist_ok=True)

        # Create subset only once per class
        if class_label not in copy_subset_images.subset_map:
            all_files = [f for f in os.listdir(src_folder) if f.startswith(class_label + "_")]
            random.shuffle(all_files)
            copy_subset_images.subset_map[class_label] = all_files[:max_per_class]

        if fname not in copy_subset_images.subset_map[class_label]:
            continue

        dst_path = os.path.join(dst_folder, fname)
        src_path = os.path.join(src_folder, fname)
        if not os.path.exists(dst_path):
            shutil.copy(src_path, dst_path)

# Keep map to track per class
copy_subset_images.subset_map = {}

#  Subsample Images (LOWER LIMITS TO PREVENT CRASHING)
import os, shutil
import random

def subsample_images(base_dir, max_images_per_class):
    for class_folder in os.listdir(base_dir):
        class_path = os.path.join(base_dir, class_folder)
        if os.path.isdir(class_path):
            images = os.listdir(class_path)
            if len(images) > max_images_per_class:
                to_delete = random.sample(images, len(images) - max_images_per_class)
                for img in to_delete:
                    os.remove(os.path.join(class_path, img))

#  Apply subsampling AFTER you organize and rename images
subsample_images(train_dir, max_images_per_class=100)
subsample_images(test_dir, max_images_per_class=30)

Data Augmentation

  • crop these images from the center to make sure they all have the same size
  • apply data augmentation techniques like random cropping and shifting during training can further enhance a model's translational invariance, helping it generalize better to variations in object positons
  • Translational Invarianc
    • ability of a model (e.g., CNN's), to recognize objects in an image regardless of their position. This robustness to shifts and translations in crucial for tasks like object detection and recognition

Image Transformations

  • T/F: to do or not

Horizontal Flip (should be set to True/False)

  • left to right (mirringing pixels from one side to other) 2. Vertical Flip (should be set to True/False)
  • flip images up to down 3. Height Shift (should be between 0 and 1)
  • bigger/smaller in height 4. Width Shift (should be between 0 and 1)
  • bigger/smaller in width 5. Rotation (should be between 0 and 180)
  • angles 6. Shear (should be between 0 and 1)
  • slant image 7. Zoom (should be between 0 and 1) etc.
  • close/farther to image

Remember not to use data augmentation in the validation/test data set.

  • test data set: should not perform augmentation

  • meant to resemble real world scenario

  • MRI scans are not suppose to do these augmentation, needs to be normal/standard of MRI machine

  • resize the images by using the parameter target_size. Here we are resizing it to 224 x 224

  • VGG16 model for Transfer Learning, which takes image inputs as 224 x 224.

  • flow_from_directory, take the images from the folder inside our specified directory, and the images from one folder will belong to same class

# Function to plot both accuracy and loss
def plot_history(history, title):
    epochs = range(len(history.history['accuracy']))

    plt.figure(figsize=(14, 5))

    # Accuracy plot
    plt.subplot(1, 2, 1)
    plt.plot(epochs, history.history['accuracy'], label='train_accuracy')
    plt.plot(epochs, history.history['val_accuracy'], label='val_accuracy')
    plt.title(f'{title} - Accuracy')
    plt.xlabel('Epoch #')
    plt.ylabel('Accuracy')
    plt.legend()

    # Loss plot
    plt.subplot(1, 2, 2)
    plt.plot(epochs, history.history['loss'], label='train_loss')
    plt.plot(epochs, history.history['val_loss'], label='val_loss')
    plt.title(f'{title} - Loss')
    plt.xlabel('Epoch #')
    plt.ylabel('Loss')
    plt.legend()

    plt.tight_layout()
    plt.show()

Conclusion

  • Transfer Learning is significantly more effective for small or imbalanced medical image datasets.

  • Proper input shape, consistent data generators, and memory-aware subsampling are essential to prevent crashes in resource-limited environments like Colab.

  • Visualization of loss and accuracy curves helped diagnose overfitting and guided architectural choices.

Recommendations

  • For this liver disease classification task:

    • Use transfer learning (VGG16 or better).

    • Fine-tune only dense layers to avoid overfitting and minimize resource usage.

  • If needed, further boost performance with:

    • Class weighting or oversampling

    • Advanced augmentations

    • More efficient architectures (e.g., MobileNet for deployment)
